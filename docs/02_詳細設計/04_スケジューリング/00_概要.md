# 詳細設計：スケジューリング 概要（MVP）

## 1. 本書の目的

本書は、計画生成におけるスケジューリング処理の仕様（入力・出力・用語・前提）を定義し、
実装の再現性とテスト容易性を担保することを目的とする。

## 2. 対象範囲

- 対象：MVP の「その日の計画生成」（1 日単位）
- 対象外：
  - 複数日を跨ぐ最適化
  - 実績学習による見積もり補正
  - チーム共有・複数ユーザー

## 3. 前提

- 本システムは「提案」を返す
- スケジューラは決定論的に動作する（同一入力 → 同一出力）
- 優先度・期限・工数に基づくルールベースの割当を行う
- LLM はスケジューリングの結果を変えない（説明生成のみ）

## 4. 用語

- 作業可能時間：ユーザーが作業できる時間帯（working_hours）
- 固定予定：作業不可時間（Event）
- 空き時間：作業可能時間から固定予定を除外した時間帯
- ブロック：計画上の時間割単位（PlanBlock）
- overflow：本日に割り当てできなかったタスク
- warnings：注意情報（空き不足、期限リスクなど）
- バッファ：予定ズレ吸収のために確保する空白枠
