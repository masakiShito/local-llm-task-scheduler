# 詳細設計：画面設計 概要（MVP）

## 1. 本書の目的

本書は、MVP における画面仕様（画面の責務、項目、操作、状態、表示ルール）を定義し、実装の基準を統一することを目的とする。

## 2. 前提

- 画面の共通レイアウト・配色・ボタンルールは「基本設計：08\_画面標準仕様.md」に従う
- API の共通仕様（レスポンス形式・エラー形式）は「詳細設計：01_API 共通仕様.md」に従う
- 本 MVP は単一ユーザー前提とする

## 3. 対象画面（MVP）

| 画面 ID | 画面名                   | 目的                                       |
| ------- | ------------------------ | ------------------------------------------ |
| SCR-100 | ホーム/ダッシュボード    | 当日の状況を俯瞰し、主要機能へ遷移する     |
| SCR-200 | タスク一覧               | タスクの参照・作成・簡易更新を行う         |
| SCR-201 | タスク詳細               | タスクの参照・状態変更を行う               |
| SCR-202 | タスク作成               | タスクを新規登録する                       |
| SCR-203 | タスク編集               | タスクを更新・削除する                     |
| SCR-300 | 固定予定一覧（対象日）   | 対象日の固定予定を管理する                 |
| SCR-301 | 固定予定詳細             | 固定予定を参照する                         |
| SCR-302 | 固定予定作成             | 固定予定を新規登録する                     |
| SCR-303 | 固定予定編集             | 固定予定を更新・削除する                   |
| SCR-400 | 計画一覧                 | 計画（Plan）の一覧を参照する               |
| SCR-401 | 計画生成（条件入力）     | 対象日・作業可能時間・制約を入力し生成する |
| SCR-402 | 計画詳細（タイムライン） | 生成結果（blocks）をタイムラインで確認する |
| SCR-403 | overflow / warnings 詳細 | overflow と warnings をまとまって確認する  |

## 4. 画面設計の記載ルール

- 項目は「表示/入力」「型」「必須」「初期値」「制約」「備考」を定義する
- 操作は「トリガー」「API」「成功時」「失敗時」を定義する
- エラーは message_id に基づく表示（詳細設計：01_API 共通仕様.md）とする

## 5. 画面 ID 命名規則

- 画面 ID は `SCR-xxx`（3 桁）とする
- 100：ホーム、200：タスク、300：固定予定、400：計画
