# 基本設計書：画面標準仕様（MVP）

## 1. 本書の目的

本書は、本システムにおける画面構成・配色・UI コンポーネントの標準を定義し、
全画面で一貫した UI/UX を実現することを目的とする。

本書は、画面一覧・画面遷移図・CRUD 図と整合する前提資料であり、
フロントエンド実装時の共通ルールとして適用する。

---

## 2. 適用範囲

- 本 MVP で実装する全画面に適用する
- 特別な理由がない限り、本仕様から逸脱しない

---

## 3. レイアウト標準

### 3.1 全体構成

全画面共通で以下のレイアウト構成とする。

- ヘッダー（固定）
- サイドバー（固定）
- メインコンテンツ領域（スクロール対象）

### 3.2 ヘッダー

- 常時表示（スクロールしても固定）
- 背景は半透明 + ぼかし（ガラス調）
- 表示内容
  - アプリ名
  - 主要画面への導線（任意）
  - 右端にユーザー系アクションのプレースホルダー
- 高さ：64px
- 下部に薄い境界線 + ハイライトラインを入れる

### 3.3 サイドバー

- 常時表示（スクロール非連動）
- 表示項目
  - タスク
  - 固定予定
  - 計画
- 現在表示中の画面を視覚的に強調表示する
- 幅：248px
- 背景は Surface を使用し、境界線で区切る
- アイコン + ラベルの組み合わせを推奨する

### 3.4 メインコンテンツ

- 画面ごとの主機能を表示する領域
- 縦スクロールは本領域のみ許可する
- 最大幅を設け、極端に横に広がらないようにする（最大 1184px）
- セクションは 24〜32px の余白で区切る
- セクション先頭に短い見出しと補足文を置く

---

## 4. ビジュアルトーン

- 清潔感・集中感を両立したミニマルモダン
- 奥行きは「境界線 + 弱い影 + 透明感」で表現する
- 強い彩度はアクセントのみで使用する

---

## 5. カラー標準

### 5.1 カラーパレット

| 用途           | 名称                     | HEX     |
| -------------- | ------------------------ | ------- |
| Primary        | アクセント（インディゴ） | #4F46E5 |
| Primary Hover  | アクセント（濃）         | #4338CA |
| Primary Soft   | アクセント淡色           | #EEF2FF |
| Background     | 画面背景                 | #F7F8FC |
| Surface        | カード/パネル            | #FFFFFF |
| Surface Alt    | サブサーフェス           | #F1F3F9 |
| Border         | 枠線                     | #E6E8F0 |
| Text Primary   | 主要テキスト             | #111827 |
| Text Secondary | 補足テキスト             | #6B7280 |
| Success        | 成功                     | #16A34A |
| Warning        | 注意                     | #F59E0B |
| Danger         | 警告                     | #EF4444 |
| Focus          | フォーカスリング         | #A5B4FC |
| Gradient A     | グラデーション左         | #6366F1 |
| Gradient B     | グラデーション右         | #8B5CF6 |

### 5.2 カラー利用ルール

- Primary は主要操作（生成・保存）にのみ使用する
- Primary Soft はカード内のアクセント帯やチップに使用する
- Warning / Danger は意味を伴う表示にのみ使用する
- 背景とカードに明確なコントラストを付け、視線誘導を行う
- テキスト色は階層を明確にし、可読性を優先する

---

## 6. タイポグラフィと余白

### 6.1 タイポグラフィ

- ベースフォントサイズ：16px
- フォント：`Inter`, `Noto Sans JP`, `system-ui`, `sans-serif`
- 見出しは太字・字間を軽く詰める
- 行間：1.5 を基準とする

### 6.2 余白・角丸

- 角丸：カード 14px / ボタン 10px / チップ 999px
- 余白は 4 の倍数を基本とする（8/12/16/24/32/40）

---

## 7. ボタン標準

### 7.1 Primary Button

- 用途：主要操作（作成・生成・保存）
- 背景色：Primary
- 文字色：白
- 高さ：44px
- 角丸：10px
- シャドウ：薄いドロップシャドウ
- ホバー時は Primary Hover、押下時は軽い沈み込み

### 7.2 Secondary Button

- 用途：補助操作（戻る・キャンセル）
- 背景色：Surface
- 枠線：Border
- 文字色：Text Primary

### 7.3 Ghost Button

- 用途：軽微な操作
- 背景色：透明
- 文字色：Text Secondary
- ホバー時は Surface Alt を使用する

### 7.4 Danger Button

- 用途：削除操作
- 背景色：Danger
- 文字色：白

---

## 8. 入力フォーム標準

### 8.1 入力項目

- ラベルは必ず表示する
- 必須項目は明示する
- エラーは入力欄直下に表示する
- フォーカス時は Focus カラーのリングを表示する
- 入力欄背景は Surface、境界線は Border を基準とする

### 8.2 フォーム構成

- 1 画面 1 目的を原則とする
- 長いフォームはセクションで分割する
- セクション見出しを設定し、視線誘導する

---

## 9. 表示コンポーネント標準

### 9.1 一覧表示

- 一覧は以下のいずれかで表現する
  - テーブル
  - カード
- 操作ボタンは右端またはカード下部に配置する

### 9.2 カード表示

- 情報単位ごとにカード化する
- 影は控えめ、境界線を優先する
- カード内の余白は 16〜24px を確保する
- ヘッダーにはアクセントラインを付けてもよい

### 9.3 チップ/バッジ

- 状態の補足情報として使用する
- Background は Surface Alt、文字色は Text Secondary を基準とする
- 重要度が高い場合は Primary Soft を使用する

### 9.4 タイムライン

- 時間割は列の高さを一定にし、視認性を優先する
- 現在時刻ラインは Primary を細線で表現する

---

## 10. メッセージ・文言ルール

### 10.1 トーン

- 命令形を避ける
- 「提案」であることを明示する
- 不安を煽る表現を使用しない

### 10.2 例

- ×「修正してください」
- ○「修正できます」
- ×「失敗しました」
- ○「処理を完了できませんでした」

---

## 11. 計画（Plan）表示に関する特則

- 計画は「確定」ではなく「提案」として扱う
- overflow / warnings は明確に区別して表示する
  - warnings：Warning カラー
  - overflow：Danger カラー
- 計画理由（LLM 生成文）は補足情報として扱う

---

## 12. MVP における割り切り

- ダークモードは対応しない
- アニメーションは控えめなフェード/ホバーに限定する
- アクセシビリティ詳細対応は Phase2 以降とする

---

- 本仕様を前提に、画面項目定義・UI コンポーネント設計を行う
- フロントエンド実装では、本仕様を共通ルールとして適用する
